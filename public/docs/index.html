Introduction

Base URL: http://localhost

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Authenticating requests

This API is not authenticated.

Endpoints

GET api/user

Example request:
curl --request GET \
    --get "http://localhost/api/user" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"

const headers = {

Update the specified unit.

Example request:
curl --request PUT \
    "http://localhost/api/medical-facilities/ab/units/11" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"name\": \"euxymhlfftcocxkjawzhjg\",
    \"description\": \"Fugiat ut soluta voluptas enim dolores velit deserunt.\",
    \"specialization\": \"aqqcgqzvdrqozcekkn\",
    \"status\": \"active\"
}"

const url = new URL(
    "http://localhost/api/medical-facilities/ab/units/11"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "name": "euxymhlfftcocxkjawzhjg",
    "description": "Fugiat ut soluta voluptas enim dolores velit deserunt.",
    "specialization": "aqqcgqzvdrqozcekkn",
    "status": "active"
};

fetch(url, {
    method: "PUT",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Request      

PUT api/medical-facilities/{facility_id}/units/{id}

Headers

Content-Type      

Example: application/json

Accept      

Example: application/json

URL Parameters

facility_id   string   

The ID of the facility. Example: ab

id   integer   

The ID of the unit. Example: 11

Body Parameters

facility_id   string  optional  

The id of an existing record in the medical_facilities table.

name   string  optional  

Must not be greater than 255 characters. Example: euxymhlfftcocxkjawzhjg

description   string  optional  

Example: Fugiat ut soluta voluptas enim dolores velit deserunt.

specialization   string  optional  

Must not be greater than 255 characters. Example: aqqcgqzvdrqozcekkn

status   string  optional  

Example: active

Must be one of:
  • active
  • inactive
